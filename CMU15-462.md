# Scotty3D — Assignment Overview (English)

This document summarizes **what each numbered assignment (A0–A4) in this repository is about**, based on **README text, test file names, and `A#T#` comments** in the source. It is for readers who do **not** know CMU **15-362 / 15-662** or Scotty3D. It does **not** explain algorithms step-by-step and does **not** include student write-ups.

**Official context (from `README.md`):** **Scotty3D** is the course **3D modeling, rendering, and animation** codebase for [CMU 15-362 / 15-662 Computer Graphics](http://15362.courses.cs.cmu.edu). Public docs: [Scotty3D User Guide](https://cmu-graphics.github.io/Scotty3D-docs/guide/).

**How work is organized:** The tree uses **Assignment 0–4** (`tests/a0` … `tests/a4`), not a `lab01` naming scheme. You run the app with `./Scotty3D` and automated checks with `./Scotty3D --run-tests "A1"` (substring filter).

## Assignment 0 (`A0`) — C++ warmup & indexed geometry

### Task 2 — Debugging drills (`tests/a0/`)

Small **C++ correctness puzzles**: fix **printing**, **integer division** in filters, **iterator** usage on `std::vector`, and **logic** in nested loops counting elements across vectors. Includes example tests that always pass/fail/ignore to illustrate the harness.

### Task 3 — Pentagon mesh (`A0T3` in `geometry/util.cpp`)

Repair **`pentagon(r)`** so the **indexed triangle list** actually forms a **planar pentagon** (starter code has an inconsistent vertex/face setup). Goal: correct **vertex positions**, **normals/UVs** as provided, and **triangle indices**—understanding half-edge internals is explicitly *not* required for this fix.

**Outcomes:** comfort with the **build/test** workflow and basic **mesh data** layout before mesh editing.

## Assignment 1 (`A1`) — Transforms & software rasterization

### Task 1 — Scene transforms (`transform.cpp`, tests `test.a1.task1`)

Implement **transform math** for the scene graph: **`local_to_parent`**, **`local_to_world`**, **`world_to_local`**, **`world_to_parent`**, **`normal_to_world`**, respecting **parent chains** (translate / rotate / scale).

### Tasks 2–4 — Rasterizer core (`rasterizer/pipeline.cpp`, `test.a1.task2`–`task4`)

- **Lines** and **triangles** rasterization in **screen space**.
- **Depth buffer** with a “less” depth test.
- **Blending**: additive and **alpha / over** compositing into the framebuffer.

### Task 5 — Interpolation (`test.a1.task5`)

Perspective-correct (or as-specified) **interpolation** of attributes across triangles (used for later shading inputs).

### Task 6 — Textures (`texture.cpp`, `test.a1.task6`)

- **Bilinear** sampling on a 2D texture.
- **Mipmaps**: build **mip levels** by downsampling; **trilinear** filtering across levels.

### Task 7 — Antialiasing (`sample_pattern.cpp`, `framebuffer.cpp`, `test.a1.task7`)

**Supersampling:** multiple **sub-pixel samples** per pixel, custom **sampling patterns**, and **resolve** (average/filter) to final colors.

### Extra credit (noted in code)

**Triangle clipping** (`A1EC: clip_triangle` in `pipeline.cpp`) — clip triangles to the view frustum before rasterization.

**Outcomes:** a **CPU rasterizer** path that can shade textured triangles with depth, transparency, and MSAA-style sampling.

## Assignment 2 (`A2`) — Half-edge mesh editing

Work is split into **local** mesh operations (`halfedge-local.cpp`) and **global** mesh algorithms (`halfedge-global.cpp`), with many matching tests under `tests/a2/`.

### Required local operators (examples)

- **Flip edge**, **collapse edge**, **extrude face** (with helper position updates).
- Optional advanced ops: **dissolve** vertex/edge, **collapse face**, **weld edges**, **split edge**, **bevel** vertex/edge, **make boundary**, etc.

### Required global operators

- **Triangulate** n-gons.
- **Linear subdivision** and **Catmull–Clark** subdivision.
- Optional: **Loop** subdivision, **isotropic remeshing**, or **mesh simplification** (pick one of several advanced tracks per comments).

**Outcomes:** robust **mesh connectivity** editing on **manifold** half-edge data, plus **subdivision / remeshing** tools used in modeling.

## Assignment 3 (`A3`) — Path tracing

Implemented mainly under `src/pathtracer/` and `src/scene/` (materials, camera).

### Task 1 — Camera rays (`camera.cpp`, `test.a3.task1.sample_ray`)

Generate **primary rays** from the camera through the image plane (including **depth-of-field** parameters where applicable).

### Task 2 — Intersection (`shape.cpp`, `test.a3.task2`)

**Ray–sphere** and **ray–triangle** hits (with **barycentric**/`uv` data as required by the framework).

### Task 3 — Acceleration (`test.a3.task3`)

- **Axis-aligned bounding boxes** for triangles.
- Build and traverse a **BVH** for faster ray–mesh intersection (**build**, **hit**, optional **fuzz**/stress tests).

### Task 4 — Lambertian BSDF (`material.cpp`, `test.a3.task4`)

Evaluate, **sample**, and **pdf** for a **diffuse** (cosine-weighted) material.

### Task 5 — Specular / transmissive materials (`test.a3.task5`)

Helpers for **reflect** / **refract** / **Schlick**; **mirror** and **glass**-style BSDFs.

### Task 7 — Image-based lighting (`samplers.cpp`, `test.a3.task7.env_light.map`)

**Environment lights**: **uniform** vs **importance sampling** of an **HDR** map on a **sphere** (PDFs for sphere and image parameterization).

**Outcomes:** a **Monte Carlo path tracer** that renders **global illumination** effects with **complex meshes** and **environment lighting**.

## Assignment 4 (`A4`) — Curves, skeletons, skinning, particles

### Task 1 — Splines (`geometry/spline.cpp`, `test.a4.task1`)

- **Hermite** curve on **[0,1]**.
- **Catmull–Rom** spline evaluation.

### Task 2 — Inverse kinematics & poses (`skeleton.cpp`, `test.a4.task2`)

- **Bind** and **current** pose: **bone-to-skeleton** transforms.
- **IK**: accumulate **gradients** for handles along bone chains; **gradient-descent** step.
- Tests cover **pose** math and **gradient** correctness.

### Task 3 — Skinning (`test.a4.task3`)

- Closest point on a **bone segment** (weighting helper).
- **Bone weights** and **linear blend skinning** of vertices.

### Task 4 — Particles (`particles.cpp`, `test.a4.task4`)

**Simulate / integrate** a particle system (state update per timestep).

**Outcomes:** basic **animation** tooling: **motion along curves**, **rigged** characters with **skinning**, and **particle** effects inside the editor/renderer pipeline.

## Supporting systems (cross-cutting)

- **UI & scene I/O:** `src/scene/`, `src/platform/`, `src/util/viewer.*` — interactive editing, loading/saving, HDR framebuffer display.
- **GPU preview rasterizer:** `src/rasterizer/` — used alongside or instead of the software path depending on mode.
- **Tests:** `src/test.cpp` registers all cases; filter by substring (e.g. `"a2.l1"` for Assignment 2, local op L1).
