## 1. Overview

### 1.1 Purpose

- **Input**: FDMJ (FuDan Mini Java) source files (`.fmj`).
- **Output**: Runnable programs:
  - **ARM**: `.s` assembly → link with `libsysy32.s` → `.out` (run via QEMU).
  - **LLVM**: `.ll` IR → link with `libsysy64.ll` → `.ll.ll` (run via `lli`).

### 1.2 Architecture

- **One frontend**: FDMJ → **Tiger IR+** (intermediate representation trees).
- **Two backends**:
  - **ARM backend**: Tiger IR+ → ARM assembly (`.s`) → link → run on QEMU (ARM).
  - **LLVM backend**: Tiger IR+ → LLVM IR (`.ll`) → SSA form → link → run with `lli`.

**Legend**: **[DIY]** = student-implemented; **[Given]** = provided by the course.

## 3. Environment and Build

### 3.1 Requirements

- **OS**: Ubuntu 20.04 LTS (e.g. WSL on Windows).
- **Tools**:
  - **gcc / g++**
  - **flex**, **bison** (Lex and Yacc)
  - **CMake** ≥ 3.20, **Ninja**
  - **clang-14**, **llvm-14** (for `.ll` and `lli`)
  - **ARM cross-compiler**: `arm-linux-gnueabihf-gcc` (e.g. gcc-arm-8.2)
  - **QEMU**: `qemu-arm` for running ARM binaries

### 3.2 Install (summary)

```bash
# Basic
sudo apt-get install gcc g++ flex bison ninja-build

# LLVM 14 (Ubuntu 20)
wget -O - https://apt.llvm.org/llvm-snapshot.gpg.key | sudo apt-key add -
sudo add-apt-repository "deb http://apt.llvm.org/focal/ llvm-toolchain-focal-14 main"
sudo apt-get update && sudo apt-get install clang-14 llvm-14

# ARM toolchain: download gcc-arm-8.2-2018.11-x86_64-arm-linux-gnueabihf.tar.xz,
# extract, and add bin/ to PATH

# QEMU: build from source for arm-linux-user target

# CMake 3.20+: download from official releases and add to PATH
```

### 3.3 Build

From the project root:

```bash
make build
```

This will:

- Configure with CMake (Ninja, Release).
- Build and install into `build/`.
- Produce `build/bin/ARMCompiler` and `build/bin/LLVMCompiler`.
- Generate `build/vendor/libsysy32.s` and `build/vendor/libsysy64.ll` for linking.

## 4. Compilation Pipeline

### 4.1 Frontend (common)

1. **Lex & Yacc [DIY]**: Read `.fmj` → tokens → **AST**.
2. **Semantics [DIY]**: Type-check AST; on first error, write message to `.output` and exit.
3. **Translate [DIY]**: During type checking, build **Tiger IR+** trees (IR Trees).

### 4.2 ARM Backend (per method)

1. **Canon [Given]**: Linearize Tiger IR+, split into **basic blocks**, reorder (trace), then split again.
2. **armgen [DIY]**: Instruction selection from Tiger IR+ basic blocks to **ARM** basic blocks (plus prologue).
3. **bg [DIY]**: Build ARM **basic block graph** and perform **dead code elimination**.
4. **assemblock [Given]**: Trace schedule blocks, add function prologue/epilogue, emit instruction stream.
5. **flowgraph, liveness, ig [Given]**: Control-flow graph, **liveness analysis**, **interference graph**.
6. **regalloc [DIY]**: **Register allocation** for ARM.
7. **assem [Given]**: Emit `.s` file.
8. **Link**: `gcc` with `libsysy32.s` → `.out`; run with `qemu-arm`.

### 4.3 LLVM Backend (per method)

1. **Canon [Given]**: Same as ARM (linearize, basic blocks, trace).
2. **llvmgen [DIY]**: Instruction selection from Tiger IR+ to **LLVM IR** basic blocks (plus prologue).
3. **bg [DIY]**: Build LLVM IR basic block graph and **dead code elimination**.
4. **ssa [DIY]**: Convert to **SSA form** (phi insertion, renaming).
5. **assemblock [Given]**: Trace, prologue/epilogue, instruction stream.
6. **flowgraph, liveness, ig [Given]**: Same idea (for consistency; not strictly required for LLVM).
7. **assem [Given]**: Emit `.ll` file.
8. **Link**: `llvm-link` with `libsysy64.ll` → `.ll.ll`; run with `lli`.

### 4.4 Debug Artifacts

- `.ast` — AST (from Lex/Yacc / printast).
- `.irp` — Original IR trees.
- `.stm` — Linearized IR and basic blocks.
- `.liv` — Liveness (and related) debug info.

## 5. FDMJ Language

### 5.1 Syntax (summary)

- **Program**: `MainMethod ClassDeclList`
  - **Main**: `public int main() { VarDeclList StmList }`
  - **Classes**: `public class id [extends id] { VarDeclList MethodDeclList }`
- **Types**: `int`, `int[]`, `class id`.
- **Statements**: blocks, if/else, while, assignment, method call, return, continue, break, putint/putch/putarray, starttime/stoptime.
- **Expressions**: literals, id, this, new int[Exp], new id(), binary ops, !, -, length(Exp), getint/getch/getarray, field access, indexing, method call.
- **Comments**: `//` and `/* */`.

### 5.2 Semantics (brief)

- **Strongly and statically typed**.
- **No global variables**; only local (in methods) and class (instance) variables.
- **Single inheritance**; no cycles; subclass inherits variables and methods; no redefinition of variables; method override must keep same signature (return type and parameter types).
- **Upcasting** allowed (e.g. `Father o = new Son();`).
- **Runtime library**: `putint`, `putch`, `putarray`, `getint`, `getch`, `getarray`, `starttime`, `stoptime` (from libsysy). **malloc** is generated by the compiler (not written in FDMJ).

See **docs/FDMJ.md** for full grammar and type-checking rules.

## 6. Frontend (Lex, Parse, Semantics, Translate)

### 6.1 Lex & Yacc (hw3–hw4) [DIY]

- **lexer.lex**: Tokenize FDMJ source; set `yylval` and return token codes to the parser.
- **parser.yacc**: Grammar per FDMJ; build **AST** in parse actions; declare tokens and nonterminals, precedence.
- **Three sections** in both files: declarations, rules (between `%%`), auxiliary C code.
- **References**: docs/MiniLexYacc.md, Lex/Yacc manuals, FDMJ.md grammar.

### 6.2 Type Checking (hw5–hw6) [DIY]

- **semant.c**: Implement type checking using:
  - **Types**: `Ty_int`, `Ty_array(Ty_int)`, `Ty_name(class_id)`.
  - **Environments**: `cenv` (class_id → class entry with vtbl/mtbl), `venv` (var_id → var entry).
  - **Entries**: `E_varEntry`, `E_classEntry`, `E_methodEntry` (see env.h).
- **Phases**:
  1. Initialize cenv, venv, MAIN_CLASS, cur_class_id, cur_method_id.
  2. **Preprocess**: Build cenv; check extend acyclicity; inherit and check no variable redefinition; method override same signature.
  3. **Type check**: Check class members and main; in each method use `S_beginScope`/`S_endScope` for venv; check variable/method use and assignments (including upcasting).

### 6.3 Translate (hw7–hw8) [DIY]

- **Strategy**: Type checking in `transXXX` body (semant); IR generation in the return part via translation helpers (translate).
- **Tr_exp**: Three flavors — **Ex** (value, T_exp), **Nx** (no value, T_stm), **Cx** (condition, T_stm + true/false patch lists). Use **unEx**, **unNx**, **unCx** to convert between them.
- **Operations**: Arithmetic → T_Binop (Ex); comparison → T_Cjump (Cx); short-circuit **and**/**or** via patch lists; **not** by swapping true/false labels.
- **Control**: if/else and while by filling patch lists for labels; maintain **whiletests** and **whileends** stacks for continue/break.
- **Arrays**: Extra word for length (e.g. at -1); malloc, then store length and elements; indexing by offset.
- **Classes**: Unified object layout (varoff, methoff, globaloff); **new** allocates and initializes (including method pointers for polymorphism); field/method access by offset; method call with `this` as first argument (e.g. t99).
- **Naming**: main → `main`; class method → `c_m`; first parameter of class methods = `t99` (this).

## 7. Backends

### 7.1 Canon [Given]

- **C_linearize**: Rewrite to canonical tree (no ESEQ/SEQ; CALL only inside EXP or MOVE(TEMP, …)).
- **C_basicBlocks**: Split linearized list into basic blocks (start with LABEL, end with JUMP/CJUMP).
- **C_traceSchedule**: Order blocks into traces (CJUMP followed by false label when possible).

### 7.2 Instruction Selection

- **LLVM (hw9–hw10) [DIY]**: Follow **docs/LLVM IR Mini.md** and the LLVM tiling diagram (e.g. docs/tiling_LLVM.png). Map Tiger IR+ to LLVM IR (add, sub, mul, sdiv, load, store, icmp, br, call, ret, alloca, phi, getelementptr, inttoptr/ptrtoint, sext, etc.).
- **ARM (hw13) [DIY]**: Follow **docs/ARM.md** and the ARM tiling diagram (e.g. docs/tiling_ARM.png). Map Tiger IR+ to ARM (add, sub, mov, ldr, str, cmp, b/bl/bx/blx, etc.). Use caller-saved (r0–r3, lr) and callee-saved (r4–r10, fp) conventions; handle stack frame (fp, sp).

### 7.3 Basic Block Graph and Dead Code [DIY]

- **bg.c**: Build the basic block graph from the block list. **Dead code elimination**: remove blocks (other than the entry) that have no predecessors (e.g. code after return/break that became unreachable).

### 7.4 SSA for LLVM (hw12) [DIY]

- **ssa.c**: Convert LLVM IR to SSA:
  - **init**: Build CFG, collect variables, compute def/use per block (entry block may be a dummy with a single jump so parameters are “defined” there).
  - **topo_sort**: Topological sort of the CFG (e.g. reverse postorder).
  - **liveness**: In/out sets per block (bottom-up using topo order).
  - **find_dom / find_idom**: Dominators and immediate dominator (idom) tree.
  - **find_df**: Dominance frontier.
  - **insert_phi**: Insert phi for variables at dominance frontier nodes (only where the variable is live-in).
  - **rename_var**: Rename variables (stack + repo for SSA names).

### 7.5 Liveness and Interference [Given]

- **flowgraph.c**: Build control-flow graph (nodes = instructions; edges from JUMP/CJUMP).
- **liveness.c**: Compute live in/out per node; def/use sets.
- **ig.c**: Build **interference graph** (two temporaries interfere if one is live at the other’s definition, or for move instructions special case).

### 7.6 Register Allocation for ARM (hw14) [DIY]

- **regalloc.c**: Graph coloring register allocator:
  - **Registers**: Simplify set (e.g. r0–r9), spill set (e.g. r10, ip, lr). Precolor machine registers to themselves.
  - **Build**: Use interference graph from liveness/ig.
  - **Simplify**: Repeatedly remove a node with degree < K (K = number of simplify registers) and push to stack.
  - **Potential spill**: If no such node, remove one and push to spill stack; repeat.
  - **Color**: Pop from simplify stack and assign a free color; then process spill stack (assign if possible = “optimistic”), rest are real spills.
  - **Adjust**: Map temps to registers; for spilled temps, use stack slots and insert load/store via spill registers. Prologue: push callee-saved (e.g. r4–r10, fp, lr); epilogue: pop and return (e.g. pop {…, pc}).

## 9. Running and Testing

### 9.1 Test Files

- Place FDMJ sources in **`test/`** as `testx.fmj` (e.g. `example.fmj`, `fibonacci.fmj`, `bubblesort.fmj`).

### 9.2 ARM

```bash
make runarm
```

For each `testx.fmj` in `test/`:

1. `ARMCompiler testx.s < testx.fmj` → produces `testx.s` (and debug files).
2. Link: `arm-linux-gnueabihf-gcc testx.s build/vendor/libsysy32.s -o testx.out`.
3. Run: `qemu-arm testx.out`; exit code is printed.

### 9.3 LLVM

```bash
make runllvm
```

For each `testx.fmj`:

1. `LLVMCompiler testx.ll < testx.fmj` → produces `testx.ll` (and debug files).
2. Link: `llvm-link testx.ll build/vendor/libsysy64.ll -S -o testx.ll.ll`.
3. Run: `lli testx.ll.ll`; exit code is printed.

### 9.4 Clean and Rebuild

- `make clean`: Remove generated files in `test/` (`.output`, `.ll`, `.s`, `.out`, `.ast`, `.irp`, `.stm`, `.liv`).
- `make veryclean`: clean + remove `build/`.
- `make rebuild`: veryclean + build.

## 10. Debug Outputs

When compiling e.g. `testx.fmj`:

- **testx.1.ast** — Printed AST (source + AST dump).
- **testx.2.irp** — Original Tiger IR+ per function.
- **testx.3.stm** — Linearized IR and basic blocks.
- **testx.4.liv** — Liveness (and related) debug.

First semantic error is written to **testx.output** (path may vary by tool; see ARMCompiler.c / LLVMCompiler.c).
