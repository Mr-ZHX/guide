# homework_spring2026 — Assignments Overview (English)

This document summarizes **what each homework bundle in this repository is for**, based on **folder names, README titles, and skeleton `TODO` markers** in the code. It targets readers who are **not** familiar with the course. It does **not** give implementation recipes and does **not** include student report text.

**Naming:** Work is organized as **`hw1` … `hw5`** (no `lab01` folders). Two optional **final project** trees are also present.

**Common tooling (from READMEs):** Python **`uv`** for environments, **Weights & Biases** for logging, and often **Modal** for cloud GPU jobs.

---

## `hw1` — Imitation learning (Push-T)

**Stated theme:** **Imitation learning** on the **Push-T** contact-rich manipulation task (`README`: “Homework 1: Imitation Learning”).

**Code focus (`src/hw1_imitation/`):** Load **Push-T** zarr data, train policies that output **action chunks** from state. Students implement:

- **`MSEPolicy`** — supervised **MSE** over predicted action chunks.
- **`FlowMatchingPolicy`** — **flow-matching** style policy with configurable **denoising steps** at inference.

**Outcomes:** train/evaluate with **`train.py`**, log to **wandb**, optional **Modal** remote run (`modal_train.py`). The skeleton leaves core **`compute_loss`** / **`sample_actions`** logic to the student.

---

## `hw2` — Policy gradients

**Stated theme:** **Policy gradient** reinforcement learning (`README`: “Homework 2: Policy Gradients”).

**Code focus:** `pg_agent.py` implements (with TODOs) **REINFORCE-style** training: trajectory processing, **advantages** (with optional **value baseline**, **reward-to-go**, **GAE**), **advantage normalization**, and policy/value updates. **`run.py`** drives **Gym** environments (e.g. CartPole example in README).

**Outcomes:** train on **CPU** or **Modal**; compare behaviors across environments and hyperparameters (batch size, iterations, etc.).

---

## `hw3` — Deep Q-learning and actor–critic (DQN + SAC)

**Stated theme:** **CS285 HW3** style content — **DQN** for **discrete** actions and **SAC** for **continuous** actions (`README`).

**Code focus:**

- **`dqn_agent.py`** — ε-greedy action selection, **target networks**, **Double-DQN**-style target action choice, critic updates.
- **`sac_agent.py`** — **twin critics**, **entropy** regularization, **reparameterized** Gaussian policy, **automatic temperature** (α) tuning, **soft/hard** target updates.

**Experiments:** YAML configs under `experiments/dqn/` (CartPole, LunarLander, MsPacman) and `experiments/sac/` (InvertedPendulum sanity, HalfCheetah, Hopper with different Q-backups).

**Outcomes:** run **`run_dqn.py`** / **`run_sac.py`** locally or via **Modal** scripts.

---

## `hw4` — LLM reinforcement learning

**Stated theme:** Fine-tune / align a **language model** with **policy-gradient** methods (`README`: “HW 4: LLM RL”).

**Tasks (from `hw4/` package):** **`format_copy`** (structured copying) and **`math_hard`** (harder math-style generation), trained with **`reinforce`** or **`grpo`** algorithms. Students complete TODOs in:

- **Token log-probabilities and masks** (`models/logprobs.py`)
- **REINFORCE** and **GRPO** update math (`rl/reinforce.py`, `rl/grpo.py`)
- **Rollout minibatch iteration** (`rollout/rollout_buffer.py`)
- **Advantage shaping** in `train.py` (group-relative normalization, optional z-score)

**Required runs (README):** Four prescribed **Modal** command lines (format_copy × {GRPO, REINFORCE}, math_hard × {GRPO, REINFORCE}), then bundle outputs for **Gradescope**.

**Outcomes:** end-to-end **LLM RL** training with **W&B** logging, evaluation intervals, and submission packaging.

---

## `hw5` — Offline reinforcement learning

**Stated theme:** **Offline RL** on robotic-style environments (`README`: “Homework 5: Offline RL”).

**Code focus:** Multiple offline algorithms in `src/agents/` — **`IQLAgent`** (expectile value, Q/V/actor losses), **`SACBCAgent`** (behavior-cloning-flavored actor–critic), **`FQLAgent`** (flow-based BC + Q losses, Euler steps for flow actions). Configs include **`sacbc`**, **`iql`**, **`fql`** (`src/configs/`).

**Outcomes:** run **`run.py`** with `--base_config` and **`cube-single-play-...`** env names; parallel **Modal** multi-seed jobs supported (`run_njobs.py`).

---

## `final_project_llm_rl` — Default LLM RL final project

**Stated theme:** **RLHF**-style alignment for instruction following (`README`: “CS 185/285 LLM RL Default Final Project”).

**Benchmark:** Fixed **Qwen2.5-1.5B-Instruct** base, curated **WildChat-derived** preference/generation splits under `dataset/wildchat_min4_judged_5k_v1`, evaluation assets in **`public_eval/`**. Primary success metric described as **GPT-judged win rate vs base model** (see README rules: token limits, decoding settings, no extra training data).

**Code areas:** Policy training (**GRPO**, **GSPO**, **DR-GRPO**, etc.), **reward model** training/eval, **online** / **offline** preference pipelines, submission builders, Modal training.

**Outcomes:** train within benchmark constraints and produceGradescope-compatible **policy** / **reward model** submissions per repo scripts.

---

## `final_project_offline_online` — Offline-to-online RL final project

**Layout:** Runnable code lives under **`final_project_offline_online/problem/`** (not the folder root).

**Stated theme:** Extend **offline RL** coursework to **online** fine-tuning (`problem/README`: “Offline to Online RL Final Project”), with agents such as **SAC-BC**, **FQL**, **IQL**, **QSM**, **DSRL** (see `src/configs/` and `src/agents/`).

**Outcomes:** same execution pattern as HW5 — **`run.py`** / **`modal_run.py`** on **`cube-single-play-...`** tasks with configurable **`base_config`** and experiment groups.
