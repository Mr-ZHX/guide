# FDCPU — Complete Lab Manual (English)

**Fudan University, Computer Architecture, Spring 2022**

This document is a complete English lab manual for the **FDCPU** project: a **RISC-V 64-bit** processor with a **5-stage pipeline**, optional **instruction and data caches**, and **CSR** support for exceptions and interrupts. The design is implemented in **SystemVerilog**, simulated with **Verilator** (with difftest), and can be synthesized for **Xilinx Vivado** (e.g. Basys-3).

**Course site**: https://fducslg.github.io/Arch-2022Spring-FDU/

## 1. Overview

### 1.1 Purpose

- Implement a **RISC-V 64-bit (RV64)** CPU core with:
  - **5-stage pipeline**: Fetch (F) → Decode (D) → Execute (E) → Memory/Read-Write (M) → Writeback (W).
  - **Hazard unit**: stalls (data hazard, load-use, mult/div, bus wait) and flushes (branch, exception/interrupt); **forwarding** for data hazards.
  - **CSR** subsystem for machine-level exceptions and interrupts (e.g. mstatus, mepc, mtvec, mcause, mtval).
  - Optional **I-cache** and **D-cache** (D-cache is set-associative; I-cache reuses D-cache logic).
  - **Bus interface**: instruction bus (IBus), data bus (DBus) to caches; unified **CBus** (burst-capable) to memory; optional AXI adapter.

### 1.2 Target

- **Simulation**: Verilator; **difftest** against NEMU (reference RISC-V interpreter) for correctness.
- **FPGA**: Xilinx Vivado; **Basys-3** (or similar) with block RAM and optional memory delay.

### 1.3 Lab Progression (from Makefile / readme)

- **Lab1**: Basic pipeline (e.g. RV64I), difftest with `lab1-test` / `lab1-test-a`.
- **Lab2**: Extended ISA and/or memory (e.g. `all-test-rv64i`).
- **Lab3**: Multiply/divide and/or cache (e.g. `all-test-rv64im`); cache test target `test-cache`.
- **Lab4**: Privilege and exceptions (e.g. `all-test-priv`); may run with or without difftest.

## 3. Architecture

### 3.1 Data Path Summary

- **F**: PC → instruction memory (IBus) → next PC (branch/jump from E stage) or PC+4; instruction and PC enter **regfd**.
- **D**: Decode instruction → control (op, alufunc, regwrite, regusage); sign-extend immediate; read regfile (rd1, rd2); destination register; pass to **regde**.
- **E**: ALU (and optionally multiplier/divider); branch resolution (pcjump); result selection (rst); pass to **regem**.
- **M**: Data memory (DBus): load/store, size/strobe; merge load data (read_data); pass to **regmw**.
- **W**: Writeback to regfile (wd, wa, wvalid); CSR updates; exception/interrupt handling (ex_pc, has_ex, has_int).

### 3.2 Bus Interfaces (common.sv)

- **IBus** (instruction):
  - Request: `valid`, `addr` (4-byte aligned).
  - Response: `addr_ok`, `data_ok`, `data` (instruction word).
- **DBus** (data):
  - Request: `valid`, `addr`, `size`, `strobe`, `data`.
  - Response: `addr_ok`, `data_ok`, `data`.
- **CBus** (unified cache–memory):
  - Request: `valid`, `is_write`, `size`, `addr`, `strobe`, `data`, `len`, `burst`.
  - Response: `ready`, `last`, `data`.

Strobes and sizes follow the comment in `common.sv`: data are aligned to 4 bytes; strobe indicates which bytes are valid.

### 3.3 Pipeline Registers (pipes.sv)

- **fetch_data_t**: pc, raw_instr, ex (exception type).
- **decode_data_t**: pc, raw_instr, ctl (control_t), imm, dst, ex.
- **execute_data_t**: pc, pc_nxt, raw_instr, ctl, imm, rst, srca, srcb, dst, ex.
- **memory_data_t**: pc, raw_instr, ctl, imm, rst, srca, srcb, dst, ex.
- **control_t**: op (decode_op_t), alufunc, regwrite, regusage (R0/R1/R2).
- **stall_t**: NOSTALL, STALLF, STALLE, STALLM, STALLW.
- **flush_t**: NOFLUSH, FLUSHM, FLUSHW.

## 4. Pipeline Stages

### 4.1 Fetch (01_fetch)

- **pcselect**: Chooses next PC: branch/jump target from E (dataE), exception vector (ex_pc), or PC+4; respects bubble (flush).
- **pcupdate**: Updates PC register with stall (stop) and bubble.
- **meminstr**: Sends instruction fetch request (ireq) and receives instruction (iresp); sets fetch exception (e.g. address misalign).
- **regfd**: Pipeline register F/D; holds when stop, clears on bubble.

### 4.2 Decode (02_decode)

- **decoder**: Maps opcode/func3/func7 to **decode_op_t** (e.g. ADDI, ADD, LD, SD, BEQ, JAL, CSR, MUL, DIV, ECALL, MRET) and **control_t** (alufunc, regwrite, regusage).
- **extend**: Sign-extends immediate (I, S, B, U, J).
- **decode**: Assembles decode_data_t (ctl, imm, dst, ex) from decoder and extend; uses **csr_imm** for CSR immediates.
- **regde**: Pipeline register D/E.

### 4.3 Execute (03_execute)

- **mux_pc_srca / mux_imm_srcb**: Select ALU operands (pc, imm, or forwarded rd1/rd2).
- **alu**: Arithmetic and logic (ADD, SUB, XOR, OR, AND, shifts, compares, etc.; 32-bit ops for *W).
- **multiplier / divider**: Multi-cycle MUL/MULW/DIV/DIVW/DIVU/REM/...; **done** indicates completion; pipeline may stall (STALLE) until done.
- **rstselecter**: Selects result among ALU, multiplier, divider.
- **pcjump**: Branch/jump target and taken decision (e.g. can_jump = dataE.rst[0] for branches).
- **regem**: Pipeline register E/M; stalled when bus or mult/div not ready.

### 4.4 Memory (04_memrw)

- **memdata**: Generates **dreq** (addr, size, strobe, data) for loads/stores; receives **dresp**; outputs **mdata** and **dbus_not_busy**; may set load/store address exception.
- **read_data / write_data**: Handle different load sizes (LB, LH, LW, LBU, LHU, LWU, LD) and store sizes (SB, SH, SW, SD) and alignment.
- **mux_rst_mdata**: Selects writeback value: ALU result or loaded data (mdata).
- **regmw**: Pipeline register M/W.

### 4.5 Writeback (05_writeback)

- **writeback**: Produces **wd** for regfile; integrates with CSR (e.g. mstatus.mie, mie); sets **has_ex**, **has_int**, **ex_pc** for exception/interrupt handling.

## 5. Hazard Handling

### 5.1 Flush (hazard/flush.sv)

- **bubble** (flush_t): On branch/jump taken or exception/interrupt, flush later stages (e.g. FLUSHM, FLUSHW) so wrong-path instructions do not commit.
- Inputs: dataE op (branch/jump), can_jump, dataM op, has_ex, has_int.

### 5.2 Stall (hazard/stall.sv)

- **stop** (stall_t): Pipeline stall when:
  - **Load-use**: D needs a value produced by a load in E or M → stall until value is available (then forward).
  - **Mult/div**: E has mult/div not done → STALLE (and possibly earlier stages).
  - **Bus**: Instruction or data bus not ready (ibus_not_busy / dbus_not_busy) → stall corresponding stage.
- Stalls are applied to pipeline registers (regfd, regde, regem, regmw) and PC (pcupdate).

### 5.3 Forward (hazard/forward.sv)

- **alua, alub**: Forward to E stage from M or W when the instruction in E needs a result produced by a later instruction (same register, not x0).
- Priority: M stage result, then W stage result (wd), else rd1/rd2 from regfile.

## 6. Memory and Caches

### 6.1 Top-Level Memory Path (VTop.sv)

- **core** issues **ireq** / **dreq** and receives **iresp** / **dresp**.
- If **USE_ICACHE**: **ICache** converts IBus ↔ CBus; else **IBusToCBus**.
- If **USE_DCACHE**: **DCache** converts DBus ↔ CBus; else **DBusToCBus**.
- **CBusArbiter** merges I-cache and D-cache CBus requests into a single **oreq** / **oresp** (to RAM or AXI).

### 6.2 DCache (cache/DCache.sv)

- **Config**: SET_NUM (e.g. 8), ASSOCIATIVITY (e.g. 2), WORDS_PER_LINE (e.g. 16); INDEX_BITS, OFFSET_BITS, TAG_BITS derived.
- **States**: IDLE, FLUSH, FETCH, DIRECT_FETCH (uncached region).
- **Uncached**: addr[31]==0 → DIRECT_FETCH; single-word CBus transaction.
- **Cached**: Hit → respond; miss → allocate line, fetch from CBus (burst), optionally writeback dirty line; LRU (age) per set.
- Uses **RAM_SinglePort** (or similar) for data/meta; **config.sv** DCACHE_BITS can control size.

### 6.3 ICache (cache/ICache.sv)

- Wraps **DCache** by converting **ireq** to **dreq** (`IREQ_TO_DREQ`) and **dresp** to **iresp** (`DRESP_TO_IRESP`); same cache logic, read-only.

### 6.4 RAM and Util

- **ram/**: Single/dual-port RAMs (e.g. RAM_SinglePort.sv) used by caches.
- **util/**: **IBusToCBus**, **DBusToCBus** (1-word transactions); **CBusArbiter** (priority or round-robin); **CBusToAXI** for AXI burst.

## 7. CSR and Exceptions

### 7.1 CSR Module (pipeline/csr/csr.sv)

- **csr_pkg.sv**: Defines CSR addresses (e.g. MSTATUS, MIE, MIP, MTVEC, MEPC, MCAUSE, MTVAL, MSCRATCH, MCYCLE, SATP) and **csr_regs_t** layout (mstatus, mie, mip, mtvec, mscratch, mepc, mcause, mtval, mcycle, etc.).
- **csr** block: Read port (ra = instruction bits for CSR index); write on commit from M stage (dataM); handles **trint, swint, exint** and produces **has_ex**, **has_int**, **ex_pc**; outputs **csr_imm** for CSR immediate instructions.

### 7.2 Exception Types (pipes.sv)

- **exception_t**: NOEX, INSTR_ADDR, INSTR_TYPE, LOAD_ADDR, STORE_ADDR, ENV_CALL.
- Exceptions are set in F (instr fetch), D (illegal instr), M (load/store addr); ENV_CALL for ecall. **ex_pc** is the trap handler address (e.g. mtvec).

### 7.3 Difftest Hooks (core.sv, under VERILATOR)

- **DifftestInstrCommit**: Commits instruction (valid, pc, instr, skip, wen, wdest, wdata).
- **DifftestArchIntRegState**: Exposes GPR state for comparison with NEMU.
- **skip**: 1 when commit is a device-memory access (e.g. addr[31]==0) so difftest ignores that write.

## 8. Simulation and FPGA

### 8.1 Verilator Build

- **Makefile** includes `verilate/Makefile.include`, `Makefile.verilate.mk`, `Makefile.vsim.mk`.
- **verilog** target: can copy vsrc to build or use Chisel-generated RTL (if SCALA_CODE).
- **emu** target: `make -C difftest emu` builds the emulator that runs Verilator model + difftest.
- **vsim** target: Builds Verilator model from **VTop.sv** (or SimTop); compiles C++ testbench (main, memory, testbench, etc.) and links.

### 8.2 Running Tests

- **test-lab1** / **test-lab1a**: Run emu with `ready-to-run/lab1` binaries; difftest against NEMU.
- **test-lab2**: `ready-to-run/lab2/all-test-rv64i.bin`.
- **test-lab3**: `ready-to-run/lab3/all-test-rv64im.bin`.
- **test-lab4**: `ready-to-run/lab4/all-test-priv.bin` (with or without difftest).
- **test-cache**: Build and run cache test (`make vsim`).
- **test-refcache**: Same with **REFERENCE_CACHE=1** (reference cache implementation).

### 8.3 Vivado (FPGA)

- **vivado/test1**, **test2**, **test3**: Vivado projects; **project/project_1.xpr**.
- **vivado/src**: **Basys-3** constraints (Basys-3-Master.xdc), **device.sv** / **device.svh**, **soc_top**, **simtop**, **bram_wrapper** or **dist_ram_wrapper**.
- **with_delay** / **without_delay**: Optional memory delay for testing.

## 9. Configuration and Parameters

### 9.1 config.sv (config_pkg)

- **USE_CACHE**, **USE_ICACHE**, **USE_DCACHE**: Enable caches (default 1).
- **ADD_LATENCY**: Memory latency simulation (e.g. 1).
- **AXI_BURST_NUM**: Burst length (e.g. 16).
- **ICACHE_BITS**, **DCACHE_BITS**: Cache size parameters (e.g. 3 for 2^3 sets or similar).

### 9.2 common.sv

- **XLEN = 64**, **MXLEN = 64**; **PCINIT = 0x8000_0000** (typical for Linux-style kernel).
- **LINK_REG_ID = 1** (x1 = ra).
- Instruction **instr_t** = 32 bits; address and data **addr_t**, **word_t** = 64 bits.
- **msize_t**: MSIZE1/2/4/8 bytes; **strobe_t**: 8-bit byte mask for 64-bit data.

### 9.3 Verilator-Only Code

- Many modules are wrapped in **`ifdef VERILATOR** so that only Verilator sees the full pipeline, caches, and difftest; Vivado may use a different top or subset (e.g. **mycpu_top_nodelay.sv** without cache delay).

### 10.3 Supported ISA (from decoder and pipes)

- **RV64I**: Integer (ADDI, ADD, SUB, shifts, compare, LUI, AUIPC, branch, JAL, JALR, load/store LD/LB/LH/LW/LBU/LHU/LWU, SD/SB/SH/SW).
- **RV64M**: MUL, MULW, DIV, DIVW, DIVU, DIVUW, REM, REMW, REMU, REMUW.
- **Zicsr**: CSRRW, CSRRS, CSRRC, CSRRWI, CSRRSI, CSRRCI; MRET, ECALL.
- Exceptions and interrupts use machine CSRs (mtvec, mepc, mcause, mtval, mstatus, mie, mip).
