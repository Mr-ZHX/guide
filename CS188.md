# Berkeley CS 188 — Pac-Man AI (`CS188/`) Lab Framework Overview (English)

This document describes **only what is present in this `CS188/` repository**: three Python assignment trees (`search/`, `reinforcement/`, `tracking/`). The code headers identify the **UC Berkeley Pac-Man AI projects** (standard CS 188 coursework). There is **no root `README.md`** in this clone.

**Scope note:** A full CS 188 term often includes **five** graded projects (plus a setup tutorial). **This workspace contains three**—consistent with **Project 1 (Search)**, **Project 3 (Reinforcement Learning)**, and **Project 4 (Inference / Bayes nets & HMMs)** in the [public CS 188 project overview](https://inst.eecs.berkeley.edu/~cs188/sp25/projects/). **Project 0**, **Project 2 (Games)**, and **Project 5 (Machine Learning)** are **not** included here.

This overview **does not** copy student reports or solution logic.

---

## Shared framework (all three trees)

- **Language:** Python 3 (course-standard library–centric).
- **Engine:** Shared-style **Pac-Man / gridworld** simulation (`pacman.py`, `game.py`, `layout.py`, graphics utilities, etc.—duplicated per project folder).
- **Grading locally:** Each subtree has **`autograder.py`**, **`grading.py`**, **`testClasses.py`**, and **`test_cases/`** (or equivalent) so you can run the same checks as in instruction.
- **License / attribution:** Berkeley `ai.berkeley.edu` attribution blocks appear in source files.

---

## Lab A — `search/` (Project 1: Search)

**Theme:** Implement **classical graph search** and **heuristic search** on Pac-Man navigation problems.

**What you implement (stubs marked `*** YOUR CODE HERE ***`):**

| File | Responsibility |
|------|----------------|
| **`search.py`** | **`depthFirstSearch`**, **`breadthFirstSearch`**, **`uniformCostSearch`**, and **`aStarSearch`** (with **`nullHeuristic`** and pluggable heuristics). Algorithms must return an **action sequence** that reaches the goal for the given `SearchProblem`. |
| **`searchAgents.py`** | Problem-specific agents and heuristics for richer tasks (e.g. visiting **corners**, clearing **food**, designing **admissible/consistency-aware** heuristics for **A\***). Only the sections marked for student edits should change. |

**Supporting pieces:** `SearchProblem` API, `SearchAgent` driver, `pacman.py` / layouts, optional **8-puzzle** (`eightpuzzle.py`) for non-Pac tests where provided.

**Outcome:** Pacman (or other agents) follows computed plans; autograder verifies correctness on hidden and released tests.

---

## Lab B — `reinforcement/` (Project 3: Reinforcement Learning)

**Theme:** **MDPs** and **temporal-difference** learning in gridworlds and related environments.

**What you implement (representative files):**

| File | Responsibility |
|------|----------------|
| **`valueIterationAgents.py`** | A **`ValueIterationAgent`**: run **value iteration** for a fixed number of iterations on a supplied **`MarkovDecisionProcess`**, then act greedily w.r.t. computed values. |
| **`qlearningAgents.py`** | **`QLearningAgent`** and related classes: maintain **Q-values**, choose actions (ε-greedy), **update** Q from samples, and support **approximate** Q-learning hooks where the skeleton indicates. |
| **`featureExtractors.py`** | (If used by your term’s handout) linear **feature** construction for approximate methods—this clone may or may not include open-ended edits; follow in-file instructions. |

**Environment code:** `gridworld.py`, `mdp.py`, `crawler.py`, `backend.py`, displays, etc., provide worlds and dynamics.

**Outcome:** Agents learn policies that maximize (discounted) reward; autograder checks values, policies, or learning curves per question.

---

## Lab C — `tracking/` (Project 4: Bayes Nets & HMMs / “Ghostbusters”)

**Theme:** **Probabilistic inference** for locating hidden variables (ghost positions) from **noisy observations**.

**What you implement (representative files):**

| File | Responsibility |
|------|----------------|
| **`factorOperations.py`** | **Factor** manipulation: **join** and **eliminate** operations on discrete **CPT** factors (building blocks for variable elimination). |
| **`inference.py`** | Build a **Bayes net** structure for the hunting scenario, run **inference by variable elimination**, implement **HMM**-style **filtering** / **prediction** / smoothing steps, and **particle-filter**-style updates where the skeleton specifies (question numbers appear in section headers in-file). |
| **`bustersAgents.py`** | An agent that uses your inference module to choose **actions** in the **Busters** game (e.g., where to move given beliefs). |

**Tests:** `tracking/test_cases/` holds many **`q*`** folders (questions) with `.test` / `.solution` pairs for exact checks.

**Outcome:** Belief distributions match references; games behave sensibly under noise.
