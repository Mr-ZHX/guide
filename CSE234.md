# CSE 234 (W25) — Programming Assignments Overview (English)

This document summarizes **what each assignment in `cse234-w25-PA` is about**, using **only materials present in the repository** (README files, notebook intros, file layout, and test names). It is intended for readers unfamiliar with the course. It does **not** explain how to implement anything and does **not** reproduce external or student-written reports (e.g. filled discussion or essay drafts).

**Course context (from bundled docs):** *CSE 234: Machine Learning Systems* — **Winter 2025** programming assignments **PA1–PA3** (there is no separate `lab01` layout; work is organized as **PA1 / PA2 / PA3** with **parts**).

## PA 1 — Automatic differentiation and fused ops (`pa1/`)

**Delivery style:** Mainly **Python**; primary narrative is in `CSE234_PA1.ipynb` (submission and honor-code rules are described there). **Public tests** live under `pa1/tests/`. **No GPU required** for this PA (per notebook intro).

### What you build (functional goals)

1. **Automatic differentiation core (`auto_diff.py`)**  
   Implement a **computational-graph** style autodiff system (TensorFlow-v1–like evaluation: build graph, then run forward/backward with an evaluator). Support standard **node-level** operations used in ML (examples reflected in tests: multiply/divide, **matmul**, **LayerNorm**, **ReLU**, **transpose**, **softmax**, **broadcast**, **sqrt**, **power**, etc.) with correct **forward** and **backward** behavior.

2. **Fused compound operations (`fused_ops.py`)**  
   Implement fused building blocks such as **MatMul + LayerNorm** and **MatMul + Softmax** as single `Op` types, with forward and backward tested against reference behavior (`test_fused_ops.py`, `test_fused_ops_perf.py`).

3. **End-to-end training script (`transformer.py`)**  
   Assemble a **small transformer-style** graph for **sequence classification** (e.g. on digit/image data loaded via sklearn/torchvision in the starter), including **softmax loss**, **SGD** epoch logic, and a training loop wiring **variables**, **forward graph**, **backward graph**, and **evaluator**—with many steps left as **TODO** in the skeleton.

**Outcomes:** a minimal **educational ML framework** from scratch plus **performance-aware** fused ops, validated by the provided **pytest** suite.

## PA 2 — GPU kernels + distributed communication (`pa2/`)

**Collaboration:** Up to **3** students may submit together (per `pa2/README.md`). Requires **MPI** and a machine with **≥ 8 cores** for the distributed portion. **Part 1** targets **Google Colab T4** for kernel timing.

### Part 1 — Triton matrix kernel (`matmul_triton.ipynb`)

**Goal:** Implement a **Triton** kernel for  
**D = ReLU(A × B + C)** with shapes *(M,K)×(K,N)→(M,N)*, **fp16** matmul/accumulation (required for stable benchmarking on T4). Work through **tile assignment**, **shared-memory tiling**, **register accumulators**, **fusion** of add + ReLU, and **epilogue** stores with masking; then **tune** block sizes (grid search) for **speedup** vs a reference PyTorch baseline.

**Outcomes:** working fused kernel + documented performance; grading ties completion to **speedup thresholds** on the prescribed benchmark shape and iteration count.

### Part 2 — MPI-based parallel training communication (`mpi_wrapper/`, `model/`, `data/`)

**Goal:** Implement **communication** for **data parallel** and **naive tensor (model) parallel** forward (**and backward where specified**) paths for a **transformer FC stack**, without focusing on full math of every op.

Concrete deliverables (file names from the tree):

- **`mpi_wrapper/comm.py`:** Student versions of **`myAllreduce`** and **`myAlltoall`** (optional timing bonus vs native MPI). Short reflection may be required in **`discussion2-1.txt`** (starter placeholder—**not** reproduced here).
- **`data/data_parallel_preprocess.py`:** **`split_train`** — partition batches across **DP** groups while **MP** ranks share data as specified.
- **`model/func_impl.py`:** **`get_info`** (shapes/groups for FC layers), plus **naive tensor-parallel** collectives for **W_o** forward and backward:  
  `naive_collect_forward_input`, `naive_collect_forward_output`, `naive_collect_backward_input`, `naive_collect_backward_output`.

**Tests:** `tests/test_data_split.py`, `test_get_info.py`, `test_transformer_forward.py`, `test_transformer_backward.py` (often run under `mpirun`).

**Outcomes:** working **MPI** collectives and correct **2D parallel** indexing for FC shards in a simplified transformer forward/backward communication pattern.

## PA 3 — MoE, scaling laws, speculative decoding, essay (`pa3/`)

**Collaboration:** Parts **1–3** may be done in groups of up to **3**; **Part 4** is **individual** (per `pa3/README.md`).

### Part 1 — Mixture of Experts (`part1/`)

**Goal:** Extend PA2-style thinking to **MoE** layers:

- **`MoE_TP` (tensor parallel):** shard expert weights across processes; use **`ShardedLinear`** and **all-reduce** to reconstruct full expert outputs (`moe.py`).
- **`MoE_EP` (expert parallel):** each rank owns whole experts; route/compute partial results and use **all-to-all** to exchange tokens as required.

**Extras:** extend **`benchmark.py`**; summarize TP vs EP behavior in **`analysis.md`** (starter may be empty—content is student work).

**Tests:** `part1/test_moe.py`.

---

### Part 2 — Training cost & scaling laws (`part2/`)

**Goal:**

1. **`model_training_cost_analysis_llama`** — From **`llama_7b_config.json`**, report **parameter count**, **forward TFLOPs per transformer layer**, and **peak fp16 memory** for one layer under stated assumptions (embeddings, layernorms, attention+MLP, checkpointing note in README).
2. **`get_optimal_N_D_from_cost`** — Given a **dollar budget**, **scaling law** *L(N,D)*, GPU **price/TFLOP** options, and **MFU**, choose GPU type and derive **optimal model size N** and **data size D**; then author **`my_model_config.json`** in Llama-like JSON form.
3. **Bonus:** analyze **DeepSeek-V3** from **`deepseek_v3_config.json`**; optional write-up **`part2/moe.md`**.

**Scripts:** `model_training_cost_analysis.py`, configs `llama_7b_config.json`, `deepseek_v3_config.json`, `my_model_config.json` (student-produced).

---

### Part 3 — Speculative decoding (`part3/`)

**Goal:** Implement **speculative decoding** — a small **draft** model proposes tokens; a large **target** model **verifies** in batched fashion; accept/reject to reduce latency.

Skeleton lives in **`PA3_Speculative_Decoding.ipynb`** (per README: fill **`initialize_*`**, **`generate_draft_tokens`**, **`verify_tokens_vectorized`**, **`speculative_decode`**).

**Outcomes:** correct algorithm, **≥1× speedup** vs baseline, **≥75%** draft acceptance (with higher tiers for extra credit), short **report** of benchmarks and ablations. Notebook should be **fully executed** before hand-in.

---

### Part 4 — Argumentative essay (individual)

**Goal:** ~**500-word** persuasive essay on a provided (or self-chosen) **forecast** about **LLM/AI markets or technology**; submit as **PDF** to a separate Gradescope slot. **Not** a coding task.
